# 📱 把英语看板装到你的手机上

看板已经做成了 **PWA（Progressive Web App）**——装到手机桌面以后，点开就像一个独立 APP：全屏、有图标、支持离线、再也不用记电脑 IP。

下面三个方案，从最快到最正式，挑一个就行。

---

## 方案 A · 同 WiFi 下立刻用（1 分钟，最快）

**适合**：你自己在家、在公司办公室，电脑和手机连同一个 WiFi。

### 1️⃣ 电脑端启服务（开一次就行，关了电脑就断）

在 `english-dashboard` 文件夹打开 PowerShell / 终端，运行：

```bash
python -m http.server 8080 --bind 0.0.0.0
```

> 加 `--bind 0.0.0.0` 是关键！默认只绑 127.0.0.1，手机连不上。

### 2️⃣ 手机浏览器打开

确认手机和电脑连同一个 WiFi，用手机浏览器访问：

```
http://192.168.1.7:8080
```

> 这是你电脑当前的局域网 IP。如果换了 WiFi 或重启了路由器，IP 可能变，重新 `ipconfig` 查一下就行。

### 3️⃣ 添加到主屏

- **iPhone（Safari）**：点底部中间的「分享」按钮 → 滑到「添加到主屏幕」→ 改名「英语看板」→ 添加。
- **安卓（Chrome）**：右上角「⋮」菜单 → 「安装应用」或「添加到主屏幕」。也可能页面顶上/底下弹出「安装到桌面」按钮（我在看板上也加了一个橙色按钮在导航栏里）。

**装完以后**：桌面多一个「英语看板」图标（蓝底白 E + 英语），点进去是全屏，没有浏览器地址栏，跟原生 APP 一样。

⚠️ **同 WiFi 方案的限制**：
- 出了 WiFi 范围就打不开（电脑关机 / 不在家都不行）
- 离线模式只缓存了一部分资源，第一次访问时的 Service Worker 会把文件都缓存下来，之后就算断网也能打开（但录音功能需要在线，因为依赖浏览器 API）

## 方案 B · 随时随地能用（推荐，5 分钟，免费）

**适合**：你想在地铁、客户现场、出差酒店都能随时打开学。

把看板免费托管到 GitHub Pages，拿到一个永久 HTTPS 网址。

### 步骤 1 · 创建 GitHub 仓库

1. 去 [github.com](https://github.com) 注册（如果还没有账号）
2. 新建仓库 `english-dashboard`（可以设为 Private 私有）
3. 把 `C:\Users\admin\Desktop\AI--\english-dashboard\` 整个文件夹传上去

最快的上传方式（命令行）：
```bash
cd C:\Users\admin\Desktop\AI--\english-dashboard
git init
git add .
git commit -m "init english dashboard"
git branch -M main
git remote add origin https://github.com/你的用户名/english-dashboard.git
git push -u origin main
```

### 步骤 2 · 开启 GitHub Pages

仓库页面 → Settings → Pages：
- Source：选 `Deploy from a branch`
- Branch：选 `main` / root
- 点 Save

等 1-2 分钟，页面上会显示：
```
Your site is live at https://你的用户名.github.io/english-dashboard/
```

### 步骤 3 · 手机访问 + 加到主屏

用手机打开这个 URL → 像方案 A 一样「添加到主屏幕」。

**这样做的好处**：
- ✅ 永久可用，电脑关机也能用
- ✅ HTTPS，iOS 支持完整的 PWA 功能
- ✅ 更新内容时只需要 `git push`，手机上下次打开自动拿最新版（Service Worker 会对比版本号更新缓存）

## 方案 C · Vercel 一键部署（如果不想用 GitHub）

1. 去 [vercel.com](https://vercel.com) 用邮箱注册
2. 点 `Add New Project` → `Upload` → 把 `english-dashboard` 文件夹拖进去
3. 等 30 秒 → 拿到一个 `https://english-dashboard-xxx.vercel.app` 网址
4. 手机打开 → 加到主屏

Vercel 的好处：不用会 git，网页直接拖文件就行。

---

## 🔄 更新内容后，手机上怎么拿到新版

**方案 A**（本地）：手机上打开看板 → 下拉刷新 → Service Worker 检测到新版本会自动拉取。

**方案 B/C**（云端）：`git push` 或重新上传后，同上，下拉刷新一次就是新版。

如果发现更新没生效：
1. 手机浏览器里打开一次原始 URL（不是桌面图标）
2. 清一下缓存（Safari：设置 → Safari → 清除历史记录）
3. 重新从桌面打开

或者更简单：在 `sw.js` 第 8 行把 `english-dashboard-v1` 改成 `v2`，推送后所有客户端都会拿到新版。

## ❓ 常见问题

**Q: iPhone 上加到主屏后，点进去为什么还有浏览器地址栏？**
A: 必须是 Safari 浏览器才行，Chrome for iOS 不支持。并且你的 URL 必须是 HTTPS（方案 B/C）或 localhost（方案 A 不行，因为是 http://IP，iOS 的 PWA 对此支持有限；建议用方案 B）。

**Q: 录音功能在 iPhone 上能用吗？**
A: 能用，但 iOS Safari 对 Web Speech API（语音识别）支持较差。想靠谱用录音检测，还是电脑 Chrome 最稳。手机上主要用：**方法区阅读 / 模板学习 / 术语卡片朗读 / 语法查阅 / 生成 Claude 指令粘贴到手机版 Claude**。

**Q: 离线能用吗？**
A: 第一次加载后，Service Worker 会把所有静态文件缓存。之后即使没网，也能打开看板、看方法、看模板、刷单词卡。但下面这几个功能需要网络：
- 🎙️ 录音识别（依赖 Google 语音服务）
- 🔊 TTS 朗读（部分机型离线可用 iOS/macOS 的原生语音）
- 📋 生成 Claude 指令需要粘贴到 Claude（所以要在线才有意义）

**Q: 能装到安卓平板 / iPad 上吗？**
A: 完全一样。响应式 CSS 对平板也做了适配（一列布局变两列）。

**Q: 数据会同步到其他设备吗？**
A: 目前进度（今日练习次数、单词记住/没记住）存在浏览器 localStorage，**不跨设备**。想跨设备同步需要加后端，后续可以考虑接 Supabase / Firebase。
